2021-01-24 3 min read

Title

Graph-Based Kinship Reasoning NetWork

Problem Description

Giving two face images, determing whether these two people have kinship.

Core Idea

  • Construct a Kinship Reasoning Graph.

Network Architecture

  1. ResNet-18 for extrcating image features.
  2. GNN on a kinship reasoning graph.
  3. MLP and Softmax for prediction.

Implementstion Details

  1. ResNet -18

    • Using sharing weights CNNs for feature extraction.
    • The two output vectors are feature representations on the same feature space.

    Since we use the same CNN to extract features for two images, the values of two features in the same dimension represent the comparison of one kind of kinship related information encoded in that dimension.

  2. Kinship Reasoning Graph

    • Feature Nodes (peripheral nodes): the values on the same dimension are defined as a feature node.

    We use one node in the kinship relational graph to denote the comparison of one feature dimension.

    • Super Node (central node): a central node created to connect feature nodes.

    Therefore, we create a super node that is connected to all other nodes while all other nodes are only connected to the super node.
