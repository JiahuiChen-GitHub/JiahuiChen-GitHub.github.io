Notes | Combine Label Propagation and Simple Models Out-performs Graph Neural Networks

2021-03-09

Under review as a conference paper at ICLR 2021.

Problem

Many ideas of new GNN architectures are adapted from new architectures in models for language (e.g. attention) or vision (w.g. deep CNNs) with the hopes that success will translate to graphs. However, as these models become more complex, understanding their perfoemance gains is a major challenge, and scaling them to large datasets is difficult.

Core Idea

A post-processing model is proposed for node classification. Base Prediction + Label Propagation (Correct & Smooth).

The key idea is that we expect errors in the base brediction to be positively correlated along edges in the graph.

Correct and Smooth Model

  1. Simple base prediction
  • Train a model fff to minimize ∑i∈LtLoss(f(xi),yi)\sum_{i\in L_t} Loss(f(x_i), y_i)∑i∈Lt​​Loss(f(xi​),yi​), fff is either a linear model or a shallow multi-layer perceptron.
  • From fff, a base prediction Z∈Rn×cZ \in \mathbb{R}^{n\times c}Z∈Rn×c can be obtained.

    In principle, we can use any base predictor for ZZZ, including those based on GNNs.

  1. Correcting for error in base predictions

    Errors in base prediction to be positively correlated along edges in the graph.

    An error at node iii increases the chance of a similar error at neighboring nodes of iii.

    Define an error matrix E∈Rn×cE\in \mathbb{R}^{n\times c}E∈Rn×c:

  2. Smoothing final predictions

    Set the training nodes back to their true labels and use the corrected predictions for the validation and unlabeled nodes.

    GLt=YLt,GLv,U=ZLv,U(r)G_{L_t}=Y_{L_t}, G_{L_v,U}=Z^{(r)}_{L_v,U} GLt​​=YLt​​,GLv​,U​=ZLv​,U(r)​

    Then iterate G(t+1)=(1−α)G+αSG(t)G^{(t+1)}=(1-\alpha)G+\alpha SG^{(t)}G(t+1)=(1−α)G+αSG(t) with G(0)=GG^{(0)}=GG(0)=G until to convergence to give the final prediction Y^\hat YY^.

    The motivation is that adjacent nodes in the graph are likely to have similar labels, which is expected given homophily or assortative properties of a network.

Summary

To review our pipeline, we start with a cheap base prediction ZZZ, using only node features but not the graph structure. After, we estimate errors E^\hat EE^ by propagating known errors on the training data, resulting in error-corrected predictions Z(r)=Z+E^Z^{(r)} = Z + \hat EZ(r)=Z+E^. Finally, we treat these as score vectors on unlabeled nodes, and combine them with the known labels through another LP step to produce smoothed final predictions. We refer to this general pipeline as Correct and Smooth (C&S).
